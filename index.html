Objective

You find yourself deep in a smelly, dank dungeon. The good news: you chose to be here. Legend tells of great riches buried below — enough gold to retire on — but you still have to make it back to the surface.

Gather what you can, survive what you can't, and find your way out. The rest is yours to discover.

Controls

Desktop

  • W A S D or arrow keys — move
  • Shift — sprint
  • Space — attack
  • Enter — start game from title screen

Mobile

  • Left-thumb joystick — 8-way movement
  • Right-thumb Attack button
  • Right-thumb Sprint button
  • Rotate to landscape to begin

Status bars

Three bars hover near your character. The red one is what keeps you breathing. The yellow one fuels the more strenuous things you'll find yourself doing — spend it, and you'll have to wait for it to recover. The cyan one only matters in the moments right after you've been hit — for a brief window, you can't be struck again, and the bar ticks down as that grace runs out.

Mobile customization

Mobile zoom defaults to 1.35×. If that feels cramped or too zoomed on your device, append ?zoom=<n> to the URL (any value from 0.5 to 5):

https://tfraczak.github.io/dungeon_crawler/?zoom=1.6

Your choice is remembered per device after the first load, so you only need to set it once.

About

Built in pure JavaScript and Canvas 2D — no frameworks, no runtime dependencies. Procedurally generated rooms, arc-based melee, and a web-audio sound engine rendered from scratch.

Dev Options

Player Cheats

Enemy Overrides

Leave blank to use the game default shown in each field. Enemy HP only affects newly-spawned enemies. Drop rate REPLACES each item's configured chance with the same value (only used when forced drops is blank). Forced drops BYPASSES the chance roll entirely and spawns exactly that many copies of every drop entry per kill — set 25 to make every kill drop 25 coins and 25 potions; set 0 to disable drops; leave blank to use the configured chance behavior.

Visualization

Map Variants

Cycle the painted background of the room you’re currently standing in — it loops through every variant available for that exit configuration. The room’s walls and exits don’t change. Force-next pins every newly-generated room to a specific exit configuration (any subset of D L R U, case-insensitive — e.g. dlu → DLU); leave blank for the procedural picker. This bypasses entry-direction and neighbor checks, so it can produce orphan rooms or doors into walls — dev-only.

No active room.

Win Condition

Force ladder skips both the coin threshold and the per-room roll, so the ladder appears in every newly-entered room (and in the next room you enter if you toggle it on mid-run). Ladder chance overrides the per-room spawn probability (0–1); leave blank for the configured default. Win scene pins the post-climb backdrop to a specific image; leave on Random for the production behavior.
